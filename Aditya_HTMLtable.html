Daily GitHub Workflow: Best Practices for Safe and Structured Collaboration

1. Daily Work Should Be Done on a Separate Branch (Not on main)

When you start new work each day, always create a new branch from main. This keeps the main branch clean and production-ready. By isolating changes, you can test and review code without affecting the main codebase.

git checkout -b feature-login-form

Once work is done, push the branch and create a pull request.

2. Always Check Git Status Before Committing

Before committing, run git status to see which files are modified, staged, or untracked. This prevents mistakes like committing unwanted files or missing changes.

git status
git add index.html
git commit -m "Added login form markup"

This step helps keep your commits clean and meaningful.

3. Verify Pull Request Link Before Sharing

Before you send your pull request link to the repo owner or teammate, double-check the URL. Make sure it points to the correct branch and repository, not an outdated or wrong one.

https://github.com/username/project-name/pull/42

Always test the link in a browser before sharing.
